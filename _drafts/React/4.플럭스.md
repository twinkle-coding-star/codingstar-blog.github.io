플러스

스토어(store)

- 데이터, UI 상태를 유지, 상태가 변경되면 이벤트를 발송[받는 뷰의 입장에서는 구독이 된다]
- 뷰는 데이터를 포함하는 스토어를 구독(subscribe)하여 스토어가 발송한 이벤트를 받게 되면, 자신을 다시 렌더링하게 됨.
- 스토어 접근자(getter)만 제공.



액션(action)

- 타입(고유한 이름), 페이로드[선택사항]
- 스토어는 액션을 받게되면 자신의 데이터를 업데이트 한다.



디스패처(dispatcher)

- 액션을 스토어로 전달하는 과정을 조율하는 역할
